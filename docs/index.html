Panel

A second opinion is good.
A panel of experts is better.

Fan out prompts to Claude, Gemini, Codex, and more — all at once, all independently.

$ brew install codebeauty/tap/panel click to copy

macOS only (Apple Silicon & Intel) · No runtime dependencies

Quick start

# Discover installed AI CLIs and create config
panel init

# Send a prompt to all enabled tools
panel run "review this authentication flow for security issues"

# Send to specific tools
panel run -t claude-opus,gemini-3-pro "is this migration safe?"

# Use a named group
panel run -f prompt.md --group smart

# Apply an expert for a focused review
panel run -E security "review this authentication flow"

# Use a team — each tool × each expert
panel run -T code-review "review this code"

# Pipe from stdin
echo "explain this error" | panel run

Features

Parallel dispatch

Run multiple AI CLIs simultaneously with configurable concurrency limits and process-group cleanup.

Read-only mode

Enforced, best-effort, or none. Each adapter uses the right flags.

Structured output

Responses, stderr, prompt, and JSON manifest per run.

Zero config start

panel init auto-discovers installed CLIs and writes a working config in seconds.

Expert teams

Group experts into reusable teams. One flag dispatches a cross-product of tools and perspectives.

Experts Beta

Assign expert roles to each tool. Same prompt, different perspectives — a security review, architecture review, and product review in parallel.

security
Vulnerabilities, OWASP Top 10, attack surfaces, secure coding practices
performance
Latency, memory usage, algorithmic complexity, scalability bottlenecks
architect
Maintainability, coupling, SOLID principles, extensibility, package structure
reviewer
Bugs, edge cases, readability, test coverage, best practices
devil
Challenge assumptions, find flaws in reasoning, argue the opposite position
product
User impact, acceptance criteria, prioritization, market fit, accessibility
# Apply an expert to all tools
panel run -E security "review this authentication flow"

# Or assign different experts per tool in config
panel run "review this code"   # each tool uses its configured expert

# Create your own
panel experts create my-golang-expert

6 built-in presets. Create unlimited custom experts as markdown files.

Teams

Group experts into reusable teams. --team creates a cross-product — each tool runs once per expert.

# Create a team
panel teams create code-review --experts security,architect,reviewer

# Run with a team (2 tools × 3 experts = 6 parallel runs)
panel run -t claude-opus,gemini-3-pro -T code-review "review this code"

Output structure

agents/panel/ review-auth-flow-1770676882/ prompt.md # original prompt (without expert) run.json # manifest with metadata summary.md # heuristic summary claude-opus.md # Claude's response claude-opus.stderr # Claude's stderr claude-opus.prompt.md # per-tool prompt with expert gemini-3-pro.md # Gemini's response gemini-3-pro.stderr # Gemini's stderr # With --team, output uses tool@expert IDs: claude-opus@security.md # Claude as security expert claude-opus@security.prompt.md # per-tool prompt

Why panel?

Inspired by counselors by Aaron Francis, rewritten in Go.

panel counselors
Runtime deps None Node.js 20+
Install size ~3 MB ~102 MB
Startup ~4 ms ~120 ms
Peak memory ~5 MB ~65 MB

Both dispatch to the same AI CLIs — response times depend on the AI provider, not the tool.
